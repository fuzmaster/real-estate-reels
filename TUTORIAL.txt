----------------------------------------------------------------
  1. WHAT THIS APP DOES
----------------------------------------------------------------

Genera Reels Web is a web interface that runs entirely on your
own computer. It gives you a form to fill in all the details
for a Spotify promotional reel — audio, artwork, stock footage,
copy — and then uses Remotion to render the finished MP4 files.

You access it through your browser at http://localhost:3000.
Nothing is sent to the internet. Everything stays on your
machine.

Everything you need is in this one folder. The Remotion video
engine is bundled inside the "remotion/" subfolder — there is
no separate project to clone or configure.


----------------------------------------------------------------
  2. REQUIREMENTS
----------------------------------------------------------------

  Node.js — the only thing you need to install yourself.
  Download the LTS version from: https://nodejs.org
  Run the installer and keep all default options.
  You only need to do this once per machine.

That's it. No Dropbox folder. No separate Remotion project.
No .env file to edit. Everything else is set up automatically.


----------------------------------------------------------------
  3. FIRST-TIME SETUP — WINDOWS
----------------------------------------------------------------

STEP 1: Double-click start.bat

  Windows may show a security warning the first time:
  click "More info" then "Run anyway".

  A terminal window opens and installs dependencies
  automatically — both the web app and the bundled Remotion
  engine. This takes 1–3 minutes the first time.
  You'll see text scrolling as packages download.

STEP 2: Open your browser

  Go to: http://localhost:3000

  The app is ready to use immediately — no setup wizard,
  no paths to configure. Your projects will be stored in
  the "remotion/public/Projects/" folder inside the app.

That's it. Future launches are just a double-click.


----------------------------------------------------------------
  4. FIRST-TIME SETUP — MAC
----------------------------------------------------------------

STEP 1: Right-click start.command → Open

  IMPORTANT: You must RIGHT-CLICK and choose Open the very
  first time. A plain double-click shows a security error
  because macOS blocks unverified scripts by default.
  After the first launch, a regular double-click works fine.

  A Terminal window opens and installs dependencies
  automatically — both the web app and the bundled Remotion
  engine. This takes 1–3 minutes the first time.

STEP 2: Open your browser

  Go to: http://localhost:3000

  The app is ready immediately — no paths to configure.

That's it. Future launches are just a double-click.


----------------------------------------------------------------
  4b. USING YOUR OWN ASSETS FOLDER (OPTIONAL)
----------------------------------------------------------------

By default, projects are stored inside the app folder at:
  remotion/public/Projects/

If you already have a Genera-Reels folder elsewhere (e.g. in
Dropbox) and want to keep using it, go to the Settings tab
(⚙ icon in the top nav) and update the paths there.
No restart needed — changes take effect immediately.


----------------------------------------------------------------
  5. STARTING THE APP EVERY DAY
----------------------------------------------------------------

Windows:  Double-click start.bat
Mac:      Double-click start.command

Your browser opens to http://localhost:3000.

To stop the server: close the terminal window, or click
inside it and press Ctrl+C.


----------------------------------------------------------------
  6. CREATING YOUR FIRST REEL
----------------------------------------------------------------

Open the "New Reel" tab. The form has these sections:

  PROJECT
  -------
  Choose "Existing Project" to pick a folder that already has
  audio and artwork in it, or "New Project" to create a fresh
  folder. Click "Create Folder" before uploading anything.

  MEDIA
  -----
  Cover Art:  Click or drag an image (JPG, PNG).
  Song Audio: Click or drag your audio file (WAV, MP3, AAC).

  STOCK FOOTAGE
  -------------
  For V1 (Driving) reels. Upload 3–5 video clips — they
  cross-fade automatically in order. Click "Pick" to reuse
  a clip already in the project. Each slot shows a thumbnail
  preview of the first frame.

  COPY
  ----
  Eyebrow:    Small text above the headline. Default "NEXT ON AUX".
  Line 1:     Large white text (the main hook).
  Line 2:     Large yellow text below it.
  Font Size:  Slider — reduce if your text is long. Default 56.
  BG Filter:  Optional CSS filter. Use presets or type your own.
  Scene Layout: Which scenes appear in the reel (see below).

    All Scenes      — Hook → Artists → Artwork (default)
    Hook + Artwork  — Hook text, then cover art.
    Artists + Artwork — Artist cascade, then cover art.
    Hook Only       — Scene 1 fills the full duration.
    Artists Only    — Scene 2 fills the full duration.
    Artwork Only    — Scene 3 (cover art) fills the full duration.

  ARTISTS
  -------
  Add each artist name and press Enter (or click Add).
  They cascade in one by one during Scene 2.
  Drag the ⠿ handle to reorder. The Punchline appears last.

  TIMING
  ------
  Duration:   Slider — how many seconds each reel runs.
  Start Times: Where in the song each reel starts. Add multiple
    to get one file per time. Formats: :32, 1:05, or 96.
  Mute Sections: Silence explicit lyrics by range (optional).

  REEL TYPES
  ----------
  V1 Driving — stock footage background
  V2 Artwork — blurred album art background
  V3 Anytime — lightweight, works without footage

  Check multiple to produce all versions in one render.

  KEYBOARD SHORTCUTS
  ------------------
  Ctrl+Enter (or ⌘+Enter on Mac) — Render Now
  Ctrl+Shift+Enter               — Add to Queue

  PREVIEW
  -------
  The phone-sized panel on the right updates live as you type.
  Switch between Hook / Artists / Artwork tabs.


----------------------------------------------------------------
  7. USING THE BATCH QUEUE
----------------------------------------------------------------

  1. Fill out a campaign, click "+ Add to Queue"
  2. Fill out the next campaign, click "+ Add to Queue"
  3. Repeat for as many campaigns as needed
  4. Go to the "Queue" tab
  5. Review — each card shows types, start times, file count
  6. Use "Duplicate" to clone a card and change one setting
  7. Click "Render All" and confirm to kick everything off


----------------------------------------------------------------
  8. WATCHING RENDERS IN THE JOBS TAB
----------------------------------------------------------------

LEFT PANEL — Job list
  - Status badge: Queued / Rendering / Done / Failed
  - Live progress bar, percentage, elapsed time, and ETA
  - "↺ Retry" button appears on failed jobs

RIGHT PANEL — Live log
  Click any job to see its full render log in real time.

A desktop notification and audio chime fire when each job
completes. Both can be toggled in the Settings tab.
Jobs persist across browser refreshes (until the tab is closed).


----------------------------------------------------------------
  9. DOWNLOADING YOUR FINISHED REELS
----------------------------------------------------------------

  1. Click the "Outputs" tab
  2. Find your campaign — files are sorted newest first by default
  3. Click "Preview" to watch inline, "Download" to save
  4. Check multiple files and click "Delete N" for bulk removal

File names: V1 - Artist Name - Part1.mp4
  V1/V2/V3 = reel type, Part N = audio start time index.


----------------------------------------------------------------
  10. TIPS AND TRICKS
----------------------------------------------------------------

  - The preview updates live — check line lengths before
    rendering. Reduce Font Size if text overflows.

  - 4 stock clips is the sweet spot for a 20-second reel.
    Keep individual clips at least 6–7 seconds to avoid
    visible loops.

  - "Sync stock footage to audio" is only useful when your
    stock footage is the music video itself.

  - Mute section timestamps are relative to the full audio
    file, not the reel clip. Add a half-second buffer on
    each side for a cleaner fade.

  - Use Export / Import buttons at the top of the form to
    save and share campaign settings as a JSON file.

  - Templates (below the Export button) save all copy and
    settings for instant reuse on future campaigns.

  - If you need to use an external assets folder (e.g. an
    existing Dropbox library), update the paths in Settings.
    Takes effect immediately, no restart needed.


----------------------------------------------------------------
  11. TROUBLESHOOTING
----------------------------------------------------------------

"Render fails at Bundling"
  The Remotion dependencies are missing. Open a terminal,
  navigate to the "remotion/" folder inside the app, and
  run: npm install

"Port 3000 already in use"
  Another instance is running. Close the other terminal.
  Or add PORT=3001 to a .env file in the app folder.

"Setup failed. Make sure Node.js is installed"
  Node.js is not installed or not on PATH. Re-download from
  nodejs.org, checking "Add to PATH" during install.

Mac: "cannot be opened because the developer cannot be verified"
  Right-click start.command → Open → Open.
  First time only.

"A render finished but the file isn't in Outputs"
  Click ↻ Refresh in the Outputs tab header.

"I moved the app folder and it stopped working"
  The bundled paths are relative, so moving the folder is
  fine. If using custom paths from Settings, update them
  there after the move.
